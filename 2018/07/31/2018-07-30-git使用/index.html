git的初始化

git配置

配置全局变量,用户名和邮箱为提交commit时的签名

`$ git config --global user.name "zysxm"` --zysxm 为github用户名
`$ git config --global user.email "451253127@qq.com" ` --github注册邮箱

配置完成后会在本地系统当前系统用户目录下创建.gitconf隐藏文件 可以输入cat/echo .gitconfig命令查看内容

$ cat ~/.gitconfig
[user]
        name = zysxm
        email = 451253127@qq.com

上面的配置文件就是Git全局配置的文件，一般配置方法是:
    git config --global <配置名称> <配置的值>
配置局部变量(切换项目路径):
    git config    命令不带--global选项来设置
此命令会在当前目录创建`.git/config`,只针对当前项目的配置

获得一个Git仓库

1. 从已有git仓库clone
2. 新建仓库,进行版本控制

Clone一个仓库

git clone 仓库地址

初始化一个新的仓库

在已存在的目录`D:/newdir`下:
    git init
git会输出:
    Initialized empty Git repository in D:newdir/.git/
输入`ls -la `出现`.git`目录,即初始化成功

正常的工作流程

1. 创建,修改
2. `git add` 添加创建或修改到本地缓存(index)
3. `git commit`提交到本地代码库
4. `git push`将本地代码库同步到远端

创建或修改文件

增删改后使用`git status`查看当前git仓库状态
文件处于`untarcked`状态时,使用`git add`加入缓存(index)

使用git add加入缓存区

将所有修改添加缓存区:
    git add . 
添加:
    git add newfile
查看状态:
    git status
此时已为`commit`做好准备
使用`git diff`加上`--cached`参数,查看缓存区那些文件被修改,按`q`退出
    git diff --cached
不加`--cached`参数时,显示当前已做的所有更改

使用git commit提交修改

加入缓存区中的文件使用`git commit`提交到本地仓库
    git commit -m "add newfile"
使新建的本地仓库关联远程仓库:
    git remote add origin 远程仓库地址
同步:
    git push origin master

分支与合并

Git的分支branch是将代码提交到主线(master)之外,同时不会影响代码库主线.

分支的作用主要体现在多人合作开发,比如A码农负责一个独立功能需要一个月时间完成,就可以创建一个分支,只把该功能的代码提交到这个分支,而其他码农仍然可以继续使用主线开发,A码农每天的提交不会对他们造成任何影响.测试通过后再将功能分支提交到主线.

创建分支

一个Git仓库可以维护多个开发分支,现在创建分支

`$ git branch my_branch`

运行git branch查看当前分支列表,以及目前开发环境处在哪个分支上:

45125@DESKTOP-I0F25I8 MINGW64 /d/Git (master)
$ git branch
* master
  my_branch

切换分支

my_branch是刚刚创建的分支,master是Git系统默认创建的主分支,*标识当前工作在哪个分支下,输入git checkout 分支名 可以切换到其他分支

45125@DESKTOP-I0F25I8 MINGW64 /d/Git (master)
$ git checkout my_branch
Switched to branch 'my_branch'
# 切换成功
45125@DESKTOP-I0F25I8 MINGW64 /d/Git (my_branch)
$

切换到my_branch分支后,编辑文件,再提交(commit)改动,最后切换回master分支

# 修改文件
$ echo 'hello git' > file1.txt

# 查看当前状态
$ git status
On branch my_branch

# 添加并提交file.text的修改
$ git add file1.txt
$ git commit -m "update file1.text"

# 查看file1.text的内容
$ cat file1.txt
hello git

# 切换分支
$ git checkout master
Switched to branch 'master'
A       file1.txt

45125@DESKTOP-I0F25I8 MINGW64 /d/Git (master)
$

这时查看file1.text的内容,已经看不到在my_branch分支下做的改动.

合并分支

在master主分支下再做一些不同的修改,然后使用merge将my_branch分支合并到master主分支:

# 切换到主分支
$ git checkout master
# 将`my_branch`合并到`master`,'-m'仍然为注释
$ git merge -m 'merge my_branch branch' my_branch
# 提交
$ git add .
$ git commit

当master主分支和my_branch分支同时更改了hello.text文件,合并显然会报错

$ git merge "merge my_branch" my_branch
merge: merge my_branch - not something we can merge

提示如下:

45125@DESKTOP-I0F25I8 MINGW64 /d/Git(master)

$ git merge "merge my_branch" my_branch
Auto-merging hello.txt
CONFLICT (content): Merge conflict in hello.txt -----提示合并冲突
Automatic merge failed; fix conflicts and then commit the result. --解决冲突并提交

此时用户显示为master|MERGING:

45125@DESKTOP-I0F25I8 MINGW64 /d/Git (master|MERGING)
$

需要手动合并冲突文件,并添加到缓存add,然后提交commit才能继续后续执行操作.

删除分支

当合并完成后,不再需要分支my_branch,就可以删除分支:

45125@DESKTOP-I0F25I8 MINGW64 /d/Git_new (master)
$ git branch -d my_branch
Deleted branch my_branch (was 4be8b88).

需注意 git branch -d 只能删除已经别合并的分支,若要强制删除某个分支,使用git branch -D

撤销一个合并

回到合并之前的状态

$ git reset --hard HEAD^

查看文件,即可发现回到合并前状态

快速向前合并

Git日志

查看日志

git log 命令显示所有的提交commit

$ git log

回车逐行显示,q退出

git log选项很多,可使用git help log查看,例:找出所有从’v2.5’开始在fs目录下的所有Makefile的修改:

$ git log v2.5.. Makefile fs/

Git会根据git log命令的参数,按时间顺序显示相关的提交(commit)

日志统计

如果使用--stat选项使用git log 会打印详细的提交记录:

$ git log --stat

此命令会显示每个提交中哪个文件被修改,这些文件添加或者修改了多少行内容

格式化日志

按自己要求格式化输出日志.--pretty 参数实现

$ git log --pretty=oneline
# 输出内容--------
a5a5ade971c5cbb55a81ad6536f68ea3827e3436 merge
637423f27ac26924df3f10b3be67e2160c300af1 456
d7380bc7f9ebb0686c1e8e905a2398593e0d14aa hehe
...

$ git log --pretty=short
# 输出内容--------
commit 4be8b88c059d6fd162b2ca7ed5911fc71b9e448a
Author: zysxm <451253127@qq.com>

    hahhah
...

其他参数medium,full,fuller,email或raw.

若都不符合自己需求,也可使用--pretty=format参数定义格式

--graph 选项可以可视化提交图(commit graph),会用ASCII字符来画出一个很很漂亮的提交历史(commit history)线:

$ git log --graph --pretty=oneline
# 输出内容
* 1a9d3e56192965adf42c374f9badf8d31d766526 (HEAD -> master) add from master
*   37ad7837625ff73d0839c5f15c1d38502fc58662 haha
|\
| * 4be8b88c059d6fd162b2ca7ed5911fc71b9e448a hahhah
* | f432d9841ac64fd7d6e3846a747b5da719547901 modify hello.text
* |   3b2b490122c7e4c93c2c691bf3e4bd51d1383239 merge
|\ \
| |/
| * d1abf7b663fc50a08fcd2b2ded20d7ea9685b1ae update text
* |   a5a5ade971c5cbb55a81ad6536f68ea3827e3436 merge
...

日志排序

日志记录可以按不同的顺序来显示,可以指定--topo-order参数,让提交按拓扑顺序来显示(就是子提交在他们的父提交前显示):

git log --pretty=format:'%h : %s' --topo-order --graph

--reverse参数逆向显示所有提交日志

小结

  • git config: 配置相关信息
  • git clone: 复制仓库
  • git init: 初始化仓库
  • git add: 添加更新内容到索引中
  • git diff: 比较内容
  • git status: 获取当前项目状态
  • git commit: 提交
  • git branch: 分支相关
  • git checkout: 切换分支
  • git merge: 合并分支
  • git reset: 恢复版本
  • git log: 查看日志
